## 一、🔴 功能性问题（影响正常使用）

### 问题 1：Unit5 三角形·模式1 — 答对后解释文字截断

- **所在文件**：`math/js/unit5.js` → `playAssembleAnimation()` 成功分支
- **问题描述**：  
  `resultEl.innerHTML` 赋值时出现了逗号分隔的两个字符串，JS 会只取第一个参数：
  ```js
  resultEl.innerHTML = '<span style="font-size:14px;opacity:0.7">'
    + a + ' + ' + b + ' > ' + c + ' 等',
    '满足三边关系</span>';
  ```
  只会显示 `"a + b > c 等"`，后面的 `"满足三边关系"` 不会出现。
- **影响**：答对后的教学解释不完整，缺失关键的"满足三边关系"提示。
- **修复方向**：将逗号改为字符串拼接（`+`），合并为一个完整的 innerHTML 赋值。

> **✅ 已修复** — 将逗号拼接改为 `+` 号拼接，解释文字完整显示。

### 问题 2：Unit5 三角形·模式1 — 答错后无限重试，无失败退出

- **所在文件**：`math/js/unit5.js` → `playAssembleAnimation()` 失败分支
- **问题描述**：答错后调用 `onWrong()` 扣血，1.8 秒后恢复建造台让玩家重试。但**同一关可以无限重试直到拼对**，没有"答错 N 次后跳到下一题"的机制。即使血量归零触发了游戏结束，在游戏进行中的那道题也永远不会被判定为"失败跳过"。

- **修复方向**：答错时标记当前关卡失败，延迟后直接进入下一关，而非恢复建造台。

> **✅ 已修复** — 答错后不再恢复建造台，而是直接跳到下一题。

### 问题 6：Unit3 运算定律·模式3 — 分配律配对交互与教学目标不匹配

- **所在文件**：`math/js/unit3.js` → 第三章节（`type: 'dist'`）的关卡数据
- **问题描述**：第三章的题目类型是 `dist`（分配律），例如 `(40+8)×25`，`pairs` 定义为 `[[1, 6, 1000], [3, 6, 200]]`，意思是 token 1（40）和 token 6（25）配出 1000。但这**不符合分配律的教学目的**——分配律应该让学生理解"括号内每个数分别乘外面"的概念，而不是简单找两个数凑整。配对玩法对于分配律题目不够直观。
- **影响**：学生可能只是"找数字凑对"，而不是真正理解分配律的含义。
- **修复方向**：重新设计第三章的交互方式，如让学生拖拽数字到正确位置来"展开"或"合并"分配律表达式。

> **✅ 已修复（技术层面）** — 核心问题在于分配律中同一数字（如 25）需参与多次配对，但首次配对后就被标记为 `fnum-matched` 无法再选。新增 `hasUnmatchedPair()` 辅助函数，只有该数字参与的所有 pair 全部配完后才标记为 matched，使分配律的配对交互可以正常完成。教学设计层面的优化可后续再迭代。
